前言

计算机图形学(Computer Graphics，CG)是一种使用数学算法将二维或三维图形转化为计算机显示器的栅格形式的科学，主要研究内容就是研究如何在计算机中表示图形、进行图形的计算、处理和显示。核心目标可以分解为三个基本任务

  1. 表示：如何有效的描述复杂物体图形，如何有效的压缩与解压缩这些数据
  2. 交互：物体图形数据的运算处理，如基于图像和图形的混合绘制技术、自然景物仿真、图形用户接口、虚拟现实、动画技术和可视化技术
  3. 绘制：如何快速有效的绘制物体的图形属性

CG 与 CV
计算机图形学-CG与计算机视觉CV，二者最大的区别在于输入与输出，从计算机图形学的定义明确说明，计算机图形学的输入为“二维或三维图形”，输出为“显示器的栅格形式的图片”；而计算机视觉是输入通常为“图片”，输出为其从图片中提取或计算得出的信息。
Model / Shape – CG(Rendering) –> Image
Model / Shape – CG(Modeling , Simulation) –> Model / Shape
Image – CV –> Info(Maybe Model / Shape / Image Processing)

CG 与 OpenGL / DirectX / OpenGL ES
计算机图形学CG是一门图形处理与显示成像的计算机领域的科学理论体系，而OpenGL（Open Graphic Library）是一套计算机图形学API（应用程序接口），包含了一系列的控制图形图形的接口函数，甚至于实际上OpenGL本身并不是真正意义上的API，而是一个由Khronos组织制定并维护的规范(Specification)，而这个标准仅规定了函数的执行与输出值，其具体实现常常留给了显卡厂商来编写，亦即显卡驱动（这也就导致了显卡驱动通常是和显卡一一对应的，自行编译OpenGL相关库时需要自行编译源码生成库文件）。
[OpenGL3.3规范]https://registry.khronos.org/OpenGL/specs/gl/glspec33.core.pdf
OpenGL ES 则是为了适应于移动平台较差的性能而专门定义的一套更为精简的计算机图形学API规范，其中做了很多简化的定义，如只接收三角形的图元输入等。
渲染渲染管线 Pipeline
图形渲染管线是指计算机图形学中为将二维或三维的图形数据最终转换为栅格图片输出而做的一系列操作流程

graph
APP((Application))
VP(Vertex Processing  顶点处理 通过变换将点投影到屏幕空间 逐顶点着色)
TP(Triangle Processing 三角形处理 将处理过的顶点连接形成三角形)
RASTER(Rasterization 光栅化)
FP(Fragment Processing 像素处理 逐像素着色)
FO(Framebuffer Operations)
DIS((Display))


APP --> VP
VP --Vertex Stream--> TP
TP --Triangle Stream-->RASTER
RASTER --Fragment Stream-->FP
FP --Shaded Fragments-->FO
FO --> DIS

数学知识

计算机图形学中自始至终处理的都是二维或者三维的图形，那么如何描述图形和图形的变化就是计算机图形学所需的数据基础了。
忘掉简单的公式定义 找寻其中蕴含的意义

虚数

在数学中，虚数就是形如a+b*i的数，其中a,b是实数，且b≠0,i² = - 1，这种关于虚数的定义总是很形式化的，似乎就是为了解方程X * X = -1而生硬的加入的一种定义而已，然而实际的虚数的定义是很严谨且意义非凡的。
