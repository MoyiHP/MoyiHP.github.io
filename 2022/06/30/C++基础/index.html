基础C程序结构

段结构

段结构
| – 程序段 ： 程序代码
| – BSS段 : 尚未初始化的全局数据
| – 数据段 ： 已初始化全局数据
| – 堆栈段 ： | – 堆区 ： 手动申请与释放
| – 栈区 ： 局部变量、函数参数与返回值 自动申请与释放
显然 BSS段、数据段属于静态内存分配区域，堆栈区属于动态内存分配区域
变量作用域
变量存在的位置与“时间” 包括 全局作用域（全局变量，常量）、局部作用域（局部变量）与形式参数
系统只针对全局变量做初始化
在当前作用域内的变量 屏蔽 更大作用域的同名变量

左值与右值
左值：指表达式结束后依然存在的持久对象，对左值可以取地址，具名变量或对象
右值：表达式结束后就不再存在的临时对象，对右值不可以取地址，没有名字

  • strcpy , strlen, strcat, strcmp 的代码实现

内存泄漏

  • 内存泄漏是指程序未能释放不再使用内存的情况，只是说这段内存失去了控制
  • 内存泄漏过多会导致内存溢出，即内存已经不再够程序使用
  • 内存泄漏分为 堆内存泄漏、系统资源泄漏、未定义析构函数为虚函数
    • 堆内存泄漏 ： 使用 malloc \ realloc \ new 等从堆中申请内存后，没有使用 free \ delete 释放
    • 系统资源泄漏 ： 使用系统资源如 bitmap \ handle \ socket 后没有释放
    • 未定义析构函数为虚函数 ： 子类析构函数无法调用父类析构函数以释放其占用的内存
  • 会导致内存泄漏的操作 ： 指针指向改变 与 未释放动态分配的内存
  • 防止内存泄漏的方法 ：将内存的分配封装在内存中，于构造函数分配内存，析构函数释放内存 - 智能指针

C与C++

C++是基于C添加了面向对象思想的语言，与C的不同主要就是在面向对象思想加入后的类的封装、继承和多态。

1. 指针与引用

指针 指向对象的地址，定义时初始化，可先后指向不同的对象
引用 相当于变量的别名，不要初始化，不能改变指代的对象

2. new delete 与 malloc free 的区别

使用格式

性质类型
new 和 delete 属于运算符，使用时变量为数据类型，返回指定数据类型的指针，执行时若数据类型为类，则还会同步执行其构造和析构函数，内存分配失败抛出异常

malloc 和 free 属于C语言的库函数，使用时参数为字节大小，返回值为（void*）指针，一般需要进行类型转换，内存分配失败返回NULL空指针

3. 调用约定 - 为啥C++可以实现函数的重载

调用约定
调用约定是对函数调用的一个约束和规定(规范)，描述了函数参数是怎么传递和由谁清除堆栈的。它决定以下内容：
(1)函数参数的压栈顺序
(2)由调用者还是被调用者把参数弹出栈
(3)产生函数修饰名的方法
典型调用约定

cdecl(C declaration，即C声明)

  • 是源起C语言的一种调用约定，也是C语言的事实上的标准
  • 函数实参在线程栈上按照从右至左的顺序依次压栈
  • 函数结果保存在寄存器EAX/AX/AL中
  • 编译后的函数名前缀以一个下划线字符
  • 调用者负责从线程栈中弹出实参（即清栈）

pascal

  • 基于Pascal语言的调用约定
  • 参数从左至右入栈(与cdecl相反)
  • 被调用者负责在返回前清理堆栈

stdcall

  • 是Windows API的标准调用约定
  • 被调用者负责清理线程栈，参数从右往左入栈。其他各方面基本与cdecl相同
  • 编译后的函数名后缀以符号”@”，后跟传递的函数参数所占的栈空间的字节长度

thiscall

  • 是C++类成员函数缺省的调用约定，但它没有显示的声明形式
  • 参数入栈：参数从右向左入栈
  • this指针入栈：如果参数个数确定，this指针通过ecx传递给被调用者；如果参数个数不确定，this指针在所有参数压栈后被压入栈 - 所以通常认为 this 指针是非静态非内联类成员函数的第一个隐式的参数。
  • 栈恢复：对参数个数不定的，调用者清理栈，否则函数自己清理栈。
    C与C++采用的调用约定 所以，C++是通过其独特的函数标识名来记录了函数的参数表，进而分辨了同一作用域内同名但不同参数表的函数，实现了函数重载

附加 C如何实现可变参数

  • 通过三个宏（va_start、va_end、va_arg）和一个类型（va_list）实现的
  • 用 va_start 获取参数列表（的地址）存储到 ap 中，用 va_arg 逐个获取值，最后用 va_arg 将 ap 置空。

数据类型 / 关键字

volatile 动态的

指定CPU每次读取变量式从内存，而非寄存器中读取
在一段程序中，一般如果两次读取同一个变量直接其值未发生改变，那么编译器会优化-将它的值放在寄存器中以加快读写，但是有些情况（比如并行程序）该变量值可能由于其他原因发生了改变，此时若编译器做了优化，后一次读取的值即为错误的。

restrict

只能用于修饰指针变量
定义该指针所指向的内存区域仅可以由该指针来做修改，防止越界操作
如memcpy(void *restrict dest, const *restrict src, size_t n)

面向对象的特性

面向对象特性包括 封装、继承和多态 ， 这些特性是通过类来实现的。

封装 继承 多态

封装

  • 将类的属性与操作捆绑在一起
  • 从而方便操作，避免外界干扰与不合理的访问

继承

  • 继承分为 实现继承、接口继承、可视继承
  • 实现继承 指 使用基类的属性和方法而无需额外编码能力
  • 接口继承 指 仅使用基类的属性和方法名称，子类必须结构实现
  • 可视继承 指 子窗口继承父窗口的外观和实现的代码

虚继承

  • 在一般多继承情况下有可能导致多个父类的占用空间与访问歧义的问题
    • 如 (B <- A ; C <- A; D <- B,C)，那么D中会有两个A的副本，占用空间，而且在访问变量a时无法确定是哪一个
  • 虚继承的含义是一个类承诺 愿意共享其父类 class C : virtual public A
  • 这样不论 基类在继承体系中出现多少次，派生类中都只保留其一份副本
  • C++的IOStream就是这样的一个实例

多态
最最重要的 运行父类指针指向子类实例

  • 多态,同一代码(函数)有着多种表现状态，多态是以封装和继承为基础的
  • 多态分为四种，分别是：
      • 静态多态指的是 函数重载overload(同一作用域，同名不同参数表的函数)、函数重写override(父子作用域，同名同参数表的函数)
    • 动态多态指的是 虚函数，更具体的指的是 使用一个指向子类实例的父类指针 调用虚函数时 实际调用的是子类中对该虚函数的实现
    • 静态多态是在静态编译期实现的，而动态多态是在动态编译期实现的

    1. 空类的大小为 1 ， 因为完全不分配内存的话实例的内存地址没办法写
    1. 当类中含有虚函数时，注意 虚指针 占用 4 字节大小

类

  • 类 ，用户定义的类型，由成员（=成员变量 + 成员函数）组成，实际是一个数据类型的蓝图(抽象)
  • 类是一个数据蓝图
    • 非静态成员变量依赖于实例而存在，每个实例拥有自己的一套非静态成员变量，在构造实例时按照定义顺序初始化，在析构实例时按定义顺序的逆序析构
      • 这里注意父类在子类中被认定为第一定义顺位
    • 非内联成员函数不依赖实例而存在，所有实例共享一套非内联成员函数，其中非静态成员函数依赖首个的默认的this指针来区别当前服务的实例，静态成员函数没有this指针但其也不访问非静态成员变量

3. 拷贝构造函数

  • 用于使用一个实例用来初始化另外一个实例，显示调用
  • 用作函数的(传值)参数，返回值，隐式调用
  • 定义形式 ：
  • 深拷贝与浅拷贝
    • 深拷贝会在堆内存中另外申请空间来储存数据，解决了指针悬挂(多个指针同时指向了同一块内存区域)的问题
    • 编译器一般会自行定义拷贝构造函数，但只是浅拷贝，这在成员变量中存在指针类型变量时不合理，因为这会导致两个实例中的指针变量同时指向了同一块内存区域
    • 为了防止浅拷贝在程序中被隐式调用导致问题，可以定义一个 私有private 的 空的 浅拷贝 构造函数，当编译器默认调用时就会报错提示

4. 析构函数

  • 一种特殊的成员函数，它会在每次删除所创建的对象时执行。
  • 析构函数的名称与类的名称是完全相同的，只是在前面加了个波浪号（~）作为前缀，它不会返回任何值，也不能带有任何参数。析构函数有助于在跳出程序（比如关闭文件、释放内存等）前释放资源。

5. 友元函数

  • 关键词 friend
  • 用于声明 友元函数、友元类
  • 友元函数可以访问类中任意成员变量，包括私有与保护级别的
    • 友元函数并不是类成员函数！其没有this指针，无法默认指定操作的实例。所以其在访问类成员时，若访问的是全局实例对象 或者 非静态实例的静态成员（函数与变量）可以直接访问；若访问的是非全局实例对象的非静态成员（函数与变量），则需要自己设立一个对象参数用来指定操作的实例。
  • 友元类 中的所有函数可以访问类中的任意成员变量

6.内联函数

  • 关键词 inline
  • 内联函数在编译(静态)时，编译器会将该函数的代码副本放置于每个调用该函数的位置，对于内联函数的修改后，需要重新编译所有涉及该函数的客户端
  • 内联函数关键词是建议性的，具体是否实现为内联函数有编译器在编译时判断
  • 内联函数需要足够简单：
    • 1.不允许使用循环语句和switch语句
    • 2.定义必须出现在内联函数第一次调用之前
    • 3.不能太长；
    • 4.最好不要递归函数
  • 内联函数并不会破坏封装的特性

  • 虚函数的实现
    • 虚函数的实现基于 虚指针 虚表 this指针
      1. 虚指针(vptr) 这是一个指向虚表首地址的指针，它是类的一个成员变量，在编译器发现类中含有虚函数时自动添加
      1. 虚表(vtable) 这是一个记录着该类中所有虚函数接口(函数描述名)与该类自己的虚函数入口地址的二维表，其存放在全局数据区
      1. 在(静态)编译时，子类首先继承父类的虚表，然后向其中添加信息，若是重写了父类的虚函数，在覆盖掉虚表中相应条目，将对应的入口地址改为自己的实现的入口地址；若不是，则向续表中添加该条信息即可
      1. 在多继承情况下，（C <- A B），子类C将拥有等同于其父类个数的虚函数表（2），同时子类C对于虚表中加入自己的修改时只会在第一个虚表中
      1. 在调用时，调用者通过this指针尝试调用虚函数，(考虑到多态，这里是父类指针指向子类实例的情况)，而this指针尽管其类型为父类但是其实际指向的是子类实例，那么在相应的内存中找到的也就是子类的vptr,于是找到的虚表也就是子类的vtable,调用的函数也就是子类的实现了。
  • 纯虚函数 通过虚函数不提供实现而加上=0实现，纯虚函数类似于接口
  • 拥有纯虚函数的类称为 抽象类，抽象类不用被实例化
常见问题

1. 虚函数能不能是内联函数 不能

  • 从编译角度看，虚函数在动态编译期实现，内联函数在静态编译期实现
  • 从实现角度看，虚函数基于 虚指针(vptr)与虚表(vtable) 和实际调用者（父类指针实际指向的实例）的类型来判断实际调用的函数实现的，而在静态编译期，编译器并不知道此时指针所指向的实例的种类，因而也就无法将该函数作为内联函数展开
    • 虚指针是类成员变量，当类中函数虚函数时编译器自动添加（这里是计算类所占空间大小的一个注意点）；虚表示放在程序段的一个表，在静态编译期生成

2.构造/析构函数 与 虚函数

构造函数能不能是虚函数 不能
能不能在构造函数中调用虚函数 能 不建议
析构函数能不能是虚函数 能 必须是
能不能在析构函数中调用虚函数 能 不建议
  • 一个子类的构造的具体顺序如下(析构时逆序)
  • 由1 、 3 可知，寻址构造函数在设置vptr之前，构造函数不能为虚函数，否则无法寻址
  • 有 3 5 6 8 可知，在执行父类构造函数体时，vptr指向父类的vtable;在执行子类构造函数体时，vptr指向子类的vtable；即此时 虚表虚指针已经退化，没有了多态的能力
  • 可以理解为
    • 派生类对象构造期间进入基类的构造函数时，对象类型变成了基类类型，而不是派生类类型。同样，进入基类析构函数时，对象也是基类类型。
    • 所以，虚函数始终仅仅调用基类的虚函数（如果是基类调用虚函数），不能达到多态的效果，所以放在构造函数中是没有意义的，而且往往不能达到本来想要的效果。
  • 析构函数则需要在调用时明确当前的析构目标，所以需要是虚函数，否则无法分辨当前析构的目标
  • 父子类是析构函数是不同名的但也是虚函数，这是C++自行实现的，析构函数甚至可以是纯虚的不过必须有定义体，因为子类的析构函数会去调用

3. 静态函数能不能是虚函数 不能

  • 静态函数先于所有类实例存在，不含有this指针

4. 构造函数能够为常函数 不能

  • 常函数只能访问而不能修改类成员变量，而构造函数的目的就是为了赋值类成员变量从而构造类实例

5 虚函数返回值的协变

  • 一般情况下，子类在override父类虚函数时，要求其返回值、函数名、参数表都必须是和父类完全一致
  • 但是在如下情况下
    • 父类的虚函数的返回值是 CBase 的引用或指针
    • 子类的虚函数的返回值是 CDerive 的引用或指针，CDerive 继承自 CBase
  • 这种情况称为“协变”，在虚函数中是允许的，且常用于 “工厂模式” 的实现中

8. 特殊函数重载

  • 以下运算符不能被重载：.、.*、::、? :、sizeof
  • 重载运算符时有三种形式 重载为成员函数 、 重载为友元函数、 重载为全局函数
    1. 重载为成员函数，第一参数必须为类实例自身，优点在于可以保证封装特性
    • 强制类型转换()、下标运算[]、指针索引运算->、赋值运算= 只能重载为成员函数
    1. 重载为全局函数，优点是第一参数可以不必为类实例，能满足一些交换律
    • 算术运算 + 、 - 和关系运算等建议为全局函数，可以满足 complex + int 和 int + complex
    1. 重载为友元函数，第一参数不必为类实例，且可以访问类中私有元素
    • 流操作符建议重载为友元函数
  • 重载运算符不改变运算符的优先级

2. 赋值运算符重载 拷贝构造函数

  • A = B 参数A是自身；参数B为被拷贝的类，应该保证其不被修改；返回值为对自身的引用

    赋值 深拷贝 与 浅拷贝

  • 赋值 即 将一个内存区域的内容拷贝到对应内存区域

  • 当对值类型做赋值时，二者内存区域相应变化，这不会有什么隐患；但是当对引用类型做赋值时，会导致两个指针指向了同一片内存区域，这会产生 “指针悬挂” “非法访问” “数据异常”等一系列问题。

  • 浅拷贝 指 仅对数据做简单赋值，不考虑其是否为值类型与引用类型

  • 深拷贝 指 对于值类型做简单赋值，对于引用类型，开辟属于自己的内存区域，将指针所指向的内存区域的值做赋值

  • C++默认会为类生成一个拷贝构造函数，但是这只是一个浅拷贝的函数，即只是简单的执行逐字节的赋值，很明显，当类成员变量存在 引用类型 时，这会导致很多内存问题

3. 流操作符重载

重载运算符函数 应该为 类成员函数 还是 友元函数

重载流操作符

  • 流操作符 本来应该为 IOSteam 的类成员函数，但是我们无法修改库，所以实现为 类的友元函数
  • 根据平常使用 (cout << a )<< b ;可以知道，参数一是流对象，参数二是类实例，返回值是 流对象的引用
4. 重载下标运算符
  • 为了支持 a[i] = 4，返回值要是引用类型
5. 重载强制类型转换符 ()
  • 重载强制类型转换运算符时，不需要指定返回值类型，因为返回值类型是确定的，就是运算符本身代表的类型
