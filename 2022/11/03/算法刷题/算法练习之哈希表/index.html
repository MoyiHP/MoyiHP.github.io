算法练习之哈希表
2022-11-03 10:04:29 # 面试 # C++ # 算法 # 数组

题目均来自力扣网

哈希表 Hash Table

理论

  1. 定义：哈希表（Hash Table，或称散列表）是根据关键码(Key)的值而直接访问元素的数据结构

  2. 用途：用于快速判断一个元素是否出现在某个集合里
    使用枚举方法时间复杂度为 On，而使用哈希表时间复杂度为 O1

  3. 哈希函数（Hash Function）：一个能够将元素值映射成一个数值索引的函数
    例如将学生名称(string)映射到其在学生名单上的索引(index)

  4. 哈希碰撞（Hash Collision）：不同的元素值经过哈希函数的映射对应到了同一个索引的情况

  • 解决哈希碰撞的方法：
      1. 链表法：将发生碰撞的元素存储在一个链表中，在索引对应存储位置存放该链表
      • 使用链表法需要合理选择哈希表的大小，太大则太多空间浪费，太小则导致链表过长增加查找时间
      1. 线性探测法：发生碰撞时，基于某种规则在哈希表中查找剩余的可用位置以存放新的元素
      • 如：当元素A经过哈希函数映射后期望的索引为i，但是此时索引为i的位置已有其他元素，发生了哈希碰撞。此时向下查找i+1位置是否可用，如果可用则将元素A最终映射到i位置，如果不可用则继续查找i+2位置，直到找到一个可用位置。类似的查找规则还有（i, i+1, i-1, i+2, i-2,…..）（i, i+1, i+4, i+9, ….）
  1. 常见的哈希结构：数组Array，集合Set，映射Map

  • 数组
    数组是最简单的哈希结构，但是使用时需要考虑其存储连续，索引连续的特性

  • 集合

    1. set 和 multiset 底层实现为 红黑树（一种平衡二叉搜索树），这决定了其Key值有序，但是只能增加和删除，而不能修改
    1. unordered_set 底层实现为 哈希表，Key值无序，但是查找性能最优，是一般最优先选择使用的哈希结构
    1. 当需要Key值有序但其不会重复时使用set，若还需要Key值可以有重复时使用 multiset
集合 底层实现 是否有序 数值是否可以重复 能否更改数值 查询效率 增删效率
std::set 红黑树 有序 否 否 O(log n) O(log n)
std::multiset 红黑树 有序 是 否 O(logn) O(logn)
std::unordered_set 哈希表 无序 否 否 O(1) O(1)
  • 映射
    1. map 和 multimap 底层实现为 红黑树（一种平衡二叉搜索树），这决定了其Key值有序
映射 底层实现 是否有序 数值是否可以重复 能否更改数值 查询效率 增删效率
std::map 红黑树 key有序 key不可重复 key不可修改 O(logn) O(logn)
std::multimap 红黑树 key有序 key可重复 key不可修改 O(log n) O(log n)
std::unordered_map 哈希表 key无序 key不可重复 key不可修改 O(1) O(1)
  1. 哈希表法可用用于快速判断一个元素是否出现在集合中，但是其需要使用额外的空间（数据结构）来存储数据，其本质上仍旧是时空平衡，牺牲空间换取时间的编程思想的体现

1. 字母异位词

https://leetcode.cn/problems/valid-anagram/
给定两个字符串 s 和 t ，编写一个函数来判断 t 是否是 s 的字母异位词
注意：若 s 和 t 中每个字符出现的次数都相同，则称 s 和 t 互为字母异位词，1 <= s.length, t.length <= 5 * 104，s 和 t 仅包含小写字母

1.2 赎金信

https://leetcode.cn/problems/ransom-note/
给你两个字符串：ransomNote 和 magazine ，判断 ransomNote 能不能由 magazine 里面的字符构成。如果可以，返回 true ；否则返回 false 。magazine 中的每个字符只能在 ransomNote 中使用一次。

代码略 和上题类似 只是判断条件放宽到（先record magazine）可以大于0

1.3 字母异位词分组

https://leetcode.cn/problems/group-anagrams/description/
给你一个字符串数组，请你将 字母异位词 组合在一起。可以按任意顺序返回结果列表。

  1. 很显然，这里与前面的区别在于不再是需要简单的对字母做哈希处理，这里真正需要的是对字符串做哈希处理，然后使用map对其进行分组
  2. 那么这里需要的就是一个合适的哈希函数 index = hash(string) ，观察到异位词排序后所得的字符串一定是相同的，非异位词排序后得到的字符串一定是不同的，这里可以使用排序后的字符串为键值（前面的题目也可以使用比较排序后字符串是否一致的方法来做）
  3. 同样也可以借鉴前面题目的思路，对每个字符串统计其各个字母出现的次数，使用record数组来作为最后分组的键值
    ps 这里需要比较了解C++ STL容器的使用方法

1.4 寻找字符串中的字母异位词

https://leetcode.cn/problems/find-all-anagrams-in-a-string/solutions/
给定两个字符串 s 和 p，找到 s 中所有 p 的 异位词 的子串，返回这些子串的起始索引。不考虑答案输出的顺序。异位词 指由相同字母重排列形成的字符串（包括相同的字符串）。

    1. 是字母异位词的两个字符串必须长度一致
    • 所以需要 s.length >= p.length
    1. 判断两个字符串是否为字母异位词可以使用一个数据结构记录各个字符的出现次数（其中ch对应的数量为 字符串p中ch出现的次数 - 字符串sub中ch出现的次数），如果这个数据结构中记录的所有ch对应的数量为0，那么p和sub是一组子母异位词
    1. 由于是在 s 中寻找符合条件且长度固定的 sub ，可以使用滑动窗口法来完成遍历

2. 两个数组的交集

https://leetcode.cn/problems/intersection-of-two-arrays/
给定两个数组 nums1 和 nums2 ，返回 它们的交集 。输出结果中的每个元素一定是 唯一 的。我们可以 不考虑输出结果的顺序 。
1 <= nums1.length, nums2.length <= 1000
0 <= nums1[i], nums2[i] <= 1000

    1. 注意到结果中每个元素唯一（需要去重）且不考虑顺序，所以这里可以使用 unordered_set 数据结构
    1. 注意到 元素的值限定在 [0,1000]，范围不大，使用固定大小的数组可能有更好的性能

2.2 两个数组的交集II

https://leetcode.cn/problems/intersection-of-two-arrays-ii/
给你两个整数数组 nums1 和 nums2 ，请你以数组形式返回两数组的交集。返回结果中每个元素出现的次数，应与元素在两个数组中都出现的次数一致（如果出现次数不一致，则考虑取较小值）。可以不考虑输出结果的顺序。

  • 进阶问题
  1. 如果给定的数组已经排好序呢？你将如何优化你的算法？
    双指针法
  2. 如果 nums1 的大小比 nums2 小，哪种方法更优？
    采用算法一的话，首先被记录的数组应该是size较小的那个
  3. 如果 nums2 的元素存储在磁盘上，内存是有限的，并且你不能一次加载所有的元素到内存中，你该怎么办？
    采用算法一，因为算法一的策略是记录nums1，遍历nums2

3. 快乐数

https://leetcode.cn/problems/happy-number/
编写一个算法来判断一个数 n 是不是快乐数。

  • 分析这个变换过程
  • 根据题意，可能出现两种情况
      1. 变换过程中出现循环，即出现了之前出现过的数，那么这个数不是快乐数
      1. 变换过程中出现了1，那么这个数是快乐数
  • 要判定之前的变换过程中是否出现过当前数，可以采用 unordered_set

4. 两数之和

给定一个整数数组 nums 和一个整数目标值 target，请你在该数组中找出 和为目标值 target 的那 两个 整数，并返回它们的数组下标。
你可以假设每种输入只会对应一个答案。但是，数组中同一个元素在答案里不能重复出现。
你可以按任意顺序返回答案。
ps 只会存在一个有效答案
more 一个时间复杂度小于 O(n2) 的算法

  • 直接的想法是 当前遍历到元素N的时候，向后查找是否有元素 target - N 的存在
  • 逆向一下，也可以当前遍历到元素N的时候，向前查找是否有元素 target - N的存在。显然向前查找的时候是在已经遍历过的元素中寻找，这种方法会更好
  • 在已经查找过的元素中寻找是否有某个特定元素，这需要使用哈希表来加速，同时还需要得到该元素对应的下标，所以，使用 unordered_map 作为存储用数据结构
