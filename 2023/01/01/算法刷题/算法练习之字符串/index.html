1. 反转字符串

https://leetcode.cn/problems/reverse-string/
编写一个函数，其作用是将输入的字符串反转过来。输入字符串以字符数组 s 的形式给出。不要给另外的数组分配额外的空间，你必须原地修改输入数组、使用 O(1) 的额外空间解决这一问题。
双指针方法
反转字符串（字符串的本质是一个数组）可以通过双指针的方法实现，I指针从头开始，J指针从尾开始，交换二者内容然后I指针向前移动，J指针向后移动，直到I >= J即可
关于交换，交换的实现方法有：

  1. 通过 swap 库函数实现
  2. 通过临时变量 tmp 实现
  3. 通过位运算实现，要求变量非0 关于 reverse 和 swap 库函数
  4. reverse 库函数采用的是双指针的思路
  5. swap 库函数采用的是临时变量的思路

3. 替换空格

https://leetcode.cn/problems/ti-huan-kong-ge-lcof/description/
请实现一个函数，把字符串 s 中的每个空格替换成”%20”。
直接想法 辅助空间
按照题目要求，利用一个辅助字符串tmp，遍历字符串s中的每个字符，如果不为空格则将其加入到tmp中，如果是空格则将”%20”加入到空格中，最后将tmp中的内容传回s中。
这种算法的时间和空间复杂度均为ON。
就地修改
直接想法中，字符元素从原空间被传输到辅助空间中组织形成最后的结果，那么如果是直接将其传送到原空间中组织呢，当字符为非空格字符时这没有问题；但是当字符是空格字符时，由于需要新加入”%20”共计3个字符，这就会导致后面的未处理字符被覆盖掉，但是如果存储的起始位置在扫描的起始位置之前共计 2 * count(count为字符串中空格字符的总数) 个位置的话，字符覆盖的问题就能够被避免。由于字符串更容易从后面被拓展，且为了获取 count也需要对字符串做一次遍历，所以这里可以选择向后拓展字符串，并从后往前传输、处理和组织字符。

关于string类型这里涉及的使用

  1. 关于 resize 和 reverse：
    resize： 扩大string的长度，这个量可以通过size获取，如果新的长度小于原来的则截断只保留靠前的元素，如果新的长度大于原来的则在后面补充空格（' '）；这个函数会影响到迭代器begin/end等的结果
    reverse：扩大string的存储空间，这个量可以通过capacity获取，预留空间的大小必须大于原有字符串的长度。这个函数的意义在于通过一次性分配足够的预留空间，从而避免在string的扩大时产生不必要的多次的自动空间扩容
  2. 关于反转迭代器 reverse_iterator
    这是通过原有的迭代器iterator挂载修饰后得到的一种修饰器迭代器，它的特点在于反转了普通迭代器的运算方向：rbegin返回指向字符串的最后一个元素的迭代器，rend返回指向字符串第一个元素的前一个位置的迭代器，+ -运算与原来相反，+运算迭代器向前运动，-运算迭代器向后运动。
    类似的还有 const_iterator 常量迭代器，const_reverse_iterator向量反转迭代器等。

关于字符串与字符数组

  1. 字符串是若干字符组成的有限序列，是一个概念上的定义；而字符数组是一个存储元素类型为字符的数组，是字符串的一个实现方式
  2. 在C语言中，将字符串实现为一个字符数组，并定义将特殊字符\0作为字符串的结束标志
  3. 在C++语言中，将字符串实现为一个string类，使用类成员变量 size 来记录了字符串的长度，因而不再需要使用特殊字符来标志字符串的结束位置，可以在string中存储\0作为一个字符使用，显示时为不占位的字符。

单词 是由非空格字符组成的字符串。s 中使用至少一个空格将字符串中的 单词 分隔开。
返回 单词 顺序颠倒且 单词 之间用单个空格连接的结果字符串。

6. 实现strStr()

https://leetcode.cn/problems/find-the-index-of-the-first-occurrence-in-a-string/
给你两个字符串 haystack 和 needle ，请你在 haystack 字符串中找出 needle 字符串的第一个匹配项的下标（下标从 0 开始）。如果 needle 不是 haystack 的一部分，则返回 -1 。haystack 和 needle 仅由小写英文字符组成。

KMP算法 - 模式匹配算法

KMP算法 由 Knuth、Morris和Pratt三位学者发明，主要应用于 字符串匹配 问题；其主要思想是 当出现字符串不匹配时，可以知道一部分之前已经匹配过的文本内容，并利用这些信息避免从头再去做匹配
字符串匹配问题
即问题6，要求在 文本串Str 中找出 模式串 Str2，并给出首次出现的起始位置。

BF 算法（暴力算法）
遍历文本串Str，对以其每个字符开始的长度等同于Str2的子字符串subStr_i，将其与Str2做StrCmp，从而找到文本串Str中的可能存在的模式串Str2。
对于BF算法，若文本串Str长度为m，模式串长度为n，则该算法的时间复杂度为 O(mn)

前缀 后缀
对于一个字符串 Str ，其前缀为 包含其首个字符的不等于其自身的所有子字符串，后缀为 包含其尾字符的不等于其自身的所有子字符串。

相等前后缀
对于一个字符串，其前后缀中相等的子字符串为相等前后缀。
分析可知，这些相等的前后缀即为前面KMP算法思路中所要找的b1与b3，由于需要不遗漏可能的子字符串，那么我们需要找到的是最长的相等前后缀。

最长相等前后缀
对于一个字符串，其最长相等前后缀（或称“最长公共前后缀”）为其前后缀中 相等的且长度最长 的子字符串。

求取 next 数组的算法
采用 动态规划（DP） 算法的方式来实现：

  1. 记录当前字符串的长度为 i , 即 后缀串的末尾为 i ; 记录 j 为当前前缀串的末尾
    1.2 有 [0, j] 为前缀串，[i-j, i]为后缀串
  2. 若 字符串长度为1，规定其为0，即 next[0] = 0
  3. 已经计算字符串的长度为 i - 1, 当前需要计算的为 next[i]
    3.1 不难分析，当上一步计算 next[i-1]时，j最后停留在 next[i-1] 位置，即 串[0,i-1]的最长公共前后缀的前缀的末尾位置，这里多加一步处理，将j++，则其将会出现在末尾位置+1的位置 3.2 IF str[j] == str[i]
    那么可以有 [0, j] = [i-j, i]，可以记 next[i] = j+1; j++;
    3.2 IF str[j] != str[i]
    那么，需要在 [0,j] 中找到一个 [0,x] 其中]x<j；在[i-j, i-1] 中找到一个[i-x-1， i-1]；使得其二者相等，且X最大。又因为有 [0,j] == [i-j-1, i-1]，那么 [0,x] == [i-x-1, i-1] 可以置换为 [0,x] == [j-x , j]，于是可以得到 x = next[j]

next 数组的优化
由之前的讨论，在实际使用next数组时，若在字符 ch 处发生了失配，则会跳转到 next[ ch.getIndex -1 ]的位置，进行匹配。那么如果跳转到的新的位置的字符 和 ch为同一字符，则这次跳转是没有意义的，其势必再次发生跳转，故而可以对 next数组进行一次如下的优化

nextval[0] = 0
nextval[j] = IF str[j] == str[next[j-1]] THEN nextval[j] = nextval[next[j-1]] ELSE nextval[j] = next[j]
